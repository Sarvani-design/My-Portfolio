UX Audit + Redesign

The Problem

Airbnb's discovery experience has a trust and decision problem. Users browsing for stays are shown a flat grid of listings without enough context to confidently shortlist — prices are inconsistent, filter categories feel arbitrary, and package-style itinerary planning has no dedicated flow.

How might we reduce friction in Airbnb's discovery and booking flow so users can move from "browsing" to "booking" with more confidence?

"Users struggle to evaluate listings effectively because the current UI prioritises surface-level visuals (price, photos) over decision-relevant context like total cost, nearby amenities, and cancellation flexibility."

Who This Is For

Leisure travellers planning a trip — solo, couple, or group — who browse Airbnb across multiple sessions before committing. They're comparing options across different trips and need the UI to help them remember, compare, and decide.

Explore — category filters that actually filter

The original Explore tab's category row (OMG!, Beach, Pools…) is decorative as much as functional. The redesign keeps the visual categories but tightens what they surface — leading with the listing image, nightly price, distance, and dates so users can evaluate at a glance without tapping in.

Search results — total cost, not just nightly rate

One of the team's core hypotheses: showing only the nightly rate misleads users about true cost. The redesigned search card shows both — "$38 night · $48 total" — so the discovery experience is honest about what a stay actually costs before the user taps in to a listing.

Photo gallery — immersive, not a sidebar

The photo gallery was redesigned as a full-bleed immersive view — large hero image with a masonry-style grid below, so users get a real sense of the space before reading a single word of the listing description.

New Feature · Core Concept

Packages — Build Your Trip in One Flow

The biggest insight from research: travellers visiting Airbnb for a destination end up juggling 3–4 tabs. Packages brings stay + experiences into a single booking flow — pick your destination, add a stay, layer in activities, and get a full itinerary with one checkout.

Why a unified flow? Research showed travellers open Airbnb for stays but leave to book activities on Google, Viator, or GetYourGuide. Packages eliminates tab-switching by making the destination — not the listing — the unit of planning. This also unlocks a new mental model: you're not booking a room, you're booking a trip.

Airbnb UI Kit — built from scratch

Before designing any screen, the team reverse-engineered Airbnb's design system: color tokens (coral #F6475F + 5 neutrals), button states, input components, cards, icons, and typography scale. This kit made every screen consistent and sped up iteration sprints.

Process

1

Initial Hypotheses

Mapped assumptions about Airbnb's discovery pain points as a team — price confusion, filter quality, listing detail overload.

2

UI Audit

Screenshotted and annotated the existing flow, identifying where the hypothesis matched real friction in the live product.

3

UI Kit + Iterations

Built the design system first, then ran sprint iterations on each screen — multiple rounds of Explore, Search, and Detail views.

4

Hi-Fi Screens

Finalised all 5 tab screens plus the new Package booking flow with full component library applied.

Reflection

Starting with a UI Kit was the right call. Building shared components first meant everyone on the team was using the same tokens — no "slightly different red" problems halfway through. It's a habit I'd carry into every group project.

The pricing hypothesis held up. Showing total cost alongside nightly rate was our boldest change — and it's actually something Airbnb has since moved toward in their real product, which validates the direction.

What I'd do next: user test the Package booking flow specifically. It's the most novel concept and the one with the most assumptions baked in — a quick prototype test with 5 travellers would tell us whether the timeline UI makes sense or needs rethinking.
